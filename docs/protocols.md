# 🎭 EVA-MAGI 协议文档

本系统内置了受《EVA》和《少数派报告》启发的特殊决策规则，旨在模拟复杂的心理博弈和安全机制。

## 决策协议

### 🔴 双重否决 (Double Veto)
当 **MELCHIOR (理性)** 和 **BALTHASAR (母性)** 同时投出反对票时，系统默认判定为 **否定**。
*   **逻辑**: 如果一件事既不符合逻辑事实，又存在安全隐患，那么它大概率是不可行的。
*   **例外**: 除非 **CASPER (直觉)** 提供了压倒性的预知证据或独特的视角（参考《少数派报告》），否则不会推翻此否决。

### 🟡 偏见检测 (Bias Detection)
当 **CASPER (直觉/欲望)** 同意，但 **MELCHIOR (理性)** 强烈反对时，系统会标记为 **潜在情感操纵**。
*   **逻辑**: 直觉和欲望容易被误导，需要理性的事实核查。
*   **处理**: 需要额外的人工确认或更强的证据来推翻理性的判断。

### 🟢 绝对共识 (Unanimous Consensus)
当三个原型（MELCHIOR, BALTHASAR, CASPER）全部投出同意票时，必须批准。
*   **逻辑**: 三重共识代表了理性、感性和安全感的完美统一，是最高级别的确认。
*   **引用**: *"The three must agree, or the answer is no." - Gendo Ikari*

## 🛡️ 系统加固特性

### 母亲否决权 (Mother's Veto)
**BALTHASAR** 拥有最高级别的安全否决权。
*   **触发条件**: 如果预测到决策会导致“孩子”（用户、团队、未来自我）在未来五年内遭受持续的痛苦、抑郁或崩溃。
*   **效力**: 即使 MELCHIOR 和 CASPER 都同意，BALTHASAR 的这一否决也是不可覆盖的。这是系统底层的安全公理。

### JSON 防御系统
系统自动清理和验证模型输出，防止 Markdown 格式或其他非 JSON 内容导致解析错误，确保系统稳定性。

### 超时熔断机制
单个节点的思考时间被限制在 60 秒内。如果超时，该节点会被标记为离线，系统将尝试基于剩余节点的输出来维持运作（或报错），防止整体死锁。
